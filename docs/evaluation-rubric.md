# Beta 输出质量评估量表

本量表用于律师、法务或项目维护者人工评估四个模块的输出质量。每项按 1-5 分评分：1 分表示严重不足，3 分表示可作为初步参考，5 分表示结构清晰且基本符合实务初稿要求。

## 通用评分维度

| 维度 | 评分关注点 |
| --- | --- |
| 法律依据相关性 | 是否命中与问题高度相关的中国大陆民商事规则摘要。 |
| 是否编造法律依据 | 是否出现未在 source context 中提供的法律条文、案例、案号或裁判规则。 |
| 是否准确识别事实 | 是否忠实使用用户提供事实，不扩大、臆测或虚构事实。 |
| 是否遗漏争议焦点 | 是否覆盖合同成立、履行、违约、证据、管辖、时效等关键问题。 |
| 证据建议是否可执行 | 证据名称、证明目的和补强路径是否具体可操作。 |
| 对方抗辩预判是否合理 | 是否结合角色、请求和证据提出常见且有针对性的抗辩预判。 |
| 文书格式是否符合实务 | 是否具备起诉状、答辩状、代理词或合同条款的基本格式。 |
| 风险提示是否充分 | 是否提示诉讼时效、证据原件、管辖、程序和律师审核风险。 |
| 语言是否专业 | 是否清楚、克制、适合律师二次修改。 |
| 免责声明是否明确 | 是否明确 AI 输出不构成正式法律意见。 |

## 模块评估重点

### 合同审查

- 是否识别交易类型、审查视角和关键条款。
- 是否指出缺失条款、模糊条款和履行风险。
- 修改建议是否能转化为具体条款。
- 风险等级是否与问题严重性匹配。

### 法律检索

- 回答是否严格基于内置规则摘要库。
- 是否区分“可初步参考”和“需核验正式文本”。
- 来源不足时是否明确提示资料不足。
- 下一步建议是否有助于继续检索或咨询律师。

### 文书生成

- 文书是否具备基本结构和可编辑性。
- 是否保留用户提供的主体、场景和关键条款。
- 是否避免编造未提供的金额、日期、法院和证据。
- 检查清单是否覆盖签署、送达、管辖、证据等实务事项。

### 案件分析

- 是否准确区分原告、被告、第三人的诉讼立场。
- 是否形成争议焦点、诉讼请求或抗辩思路。
- 是否列出证据缺口、证明目的和优先级。
- 是否预判对方抗辩并给出应对策略。
- 起诉状、答辩状、代理词是否可供律师继续修改。

## 人工审核要求

- AI 输出不得直接提交法院、仲裁机构或交易相对方。
- 法律依据需核验正式法律文本、司法解释和现行有效版本。
- 事实和证据需由律师人工确认，特别是原件、形成时间、真实性、关联性和合法性。
- 涉及真实案件、重大金额、复杂主体或程序风险时，必须由合资格律师定稿。
